Student Behavioral Dataset

About Dataset

Student Performance & Behavior Dataset

  • This dataset is real data of 5,000 records collected from a private learning provider.
  • The dataset includes key attributes necessary for exploring patterns, correlations, and insights related to academic performance.

Columns:

  • Student_ID: Unique identifier for each student.
  • First_Name: Students first name.
  • Last_Name: Students last name.
  • Email: Contact email (can be anonymized).
  • Gender: Male, Female, Other.
  • Age: The age of the student.
  • Department: Student's department (e.g., CS, Engineering, Business).
  • Attendance (%): Attendance percentage (0-100%).
  • Midterm_Score: Midterm exam score (out of 100).
  • Final_Score: Final exam score (out of 100).
  • Assignments_Avg: Average score of all assignments (out of 100).
  • Quizzes_Avg: Average quiz scores (out of 100).
  • Participation_Score: Score based on class participation (0-10).
  • Projects_Score: Project evaluation score (out of 100).
  • Total_Score: Weighted sum of all grades.
  • Grade: Letter grade (A, B, C, D, F).
  • Study_Hours_per_Week: Average study hours per week.
  • Extracurricular_Activities: Whether the student participates in extracurriculars (Yes/No).
  • Internet_Access_at_Home: Does the student have access to the internet at home? (Yes/No).
  • Parent_Education_Level: Highest education level of parents (None, High School, Bachelor's, Master's, PhD).
  • Family_Income_Level: Low, Medium, High.
  • Stress_Level (1-10): Self-reported stress level (1: Low, 10: High).
  • Sleep_Hours_per_Night: Average hours of sleep per night.

The Attendance is not part of the Total_Score or has very minimal weight.

Calculating the weighted sum:

Total Score=aMidterm+bFinal+cAssignments+dQuizzes+eParticipation+fProjects

Dataset contains:

  • Missing values (nulls): in some records (e.g., Attendance, Assignments, or Parent Education Level).
  • Bias in some Datae (ex: grading e.g., students with high attendance get slightly better grades).
  • Imbalanced distributions (e.g., some departments having more students).

2. Pivot() method in pandas

The pivot() method in Pandas is used to reshape a DataFrame based on values from one or more columns. It takes unique values from specified columns and turns them into new columns, effectively rotating the data from "long" to "wide" format.

Syntax:

DataFrame.pivot(index=None, columns=None, values=None)

Precise Breakdown of pivot() Parameters:

  • index (required):

    • This specifies the column (or list of columns) whose unique values will form the new DataFrame's index (rows).
    • If you don't provide this, Pandas will try to use the existing index, but it's usually best to specify it.
    • Each unique combination of values in the index column(s) will become a row in the pivoted table.

  • columns (required):

    • This specifies the column (or list of columns) whose unique values will form the new DataFrame's columns.
    • Each unique value (or combination of values) in the columns column(s) will become a column header in the pivoted table.

  • values (optional):

    • This specifies the column (or list of columns) whose values will populate the new DataFrame's cells.
    • If values is not specified, pivot() will use all remaining non-index/non-columns columns as the values, and the resulting DataFrame will have a MultiIndex for its columns (level 1 for the original column names, level 2 for the new pivoted column names).
    • If a specific values column is chosen, only its data will be used to fill the cells.

Key Characteristics and How it Works:

  • Unique Combinations are Crucial: For pivot() to work correctly, the combination of values specified by index and columns must uniquely identify a single value in the values column for each cell.
    • If there are duplicate combinations of index and columns values (meaning, for a given index-column pair, there are multiple values), pivot() will raise a ValueError: Index contains duplicate entries, cannot reshape.
    • If you have duplicate combinations and need to aggregate the values (e.g., sum them, average them), you should use pivot_table() instead, which is more flexible.
  • Reshaping Data: It transforms data from a "long" format (where categories are represented as values in a column) to a "wide" format (where categories become distinct columns).
  • Missing Values: If a combination of index and columns values does not exist in the original DataFrame, NaN (Not a Number) will be placed in the corresponding cell of the pivoted table.

When to Use pivot():

  • When you have data in a "long" format and want to transform it into a "wide" format.
  • When you know that the combination of your chosen index and columns uniquely identifies the values you want in the cells.
  • To create a more readable summary table for certain types of data.

Key Insights

  • The pivot() method in Pandas is excellent for reshaping data from a "long" to a "wide" format, effectively rotating values from specific columns into new column headers while precisely retaining the original data points.
  • However, its most significant drawback, is its inability to perform aggregation on numeric data. This means that if a combination of the chosen index and columns values in your original DataFrame leads to multiple corresponding values entries, pivot() cannot resolve this ambiguity by summing, averaging, or performing any other statistical operation. Instead, it will raise an error (a ValueError indicating duplicate entries).
  • Given that a key aspect of data analysis involves grouping data and deriving summary statistics through aggregation, this limitation restricts pivot() solely to scenarios where you're reshaping data without any need to summarize or combine original values. To overcome this fundamental limitation, Pandas provides the more versatile pivot_table() method, which is specifically designed to handle both data reshaping and the aggregation of numeric data simultaneously.
